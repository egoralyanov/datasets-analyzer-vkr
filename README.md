# Интеллектуальная система анализа наборов данных для решения задач машинного обучения

Выпускная квалификационная работа бакалавра.
МГТУ им. Н.Э. Баумана, кафедра ИУ5.

## Описание

Веб-приложение для первичной диагностики табличных датасетов перед задачами машинного обучения. Принимает CSV/XLSX, выполняет EDA-профилирование, выявляет проблемы качества данных, рекомендует тип ML-задачи (бинарная / многоклассовая классификация, регрессия, кластеризация) с обоснованием, подбирает похожие наборы данных из каталога и формирует PDF-отчёт. Поддерживает историю анализов и базовую административную панель.

## Состав системы

- **Профайлер** — 33 meta-признака датасета: типы и кардинальность колонок, статистики распределений (skewness, нормальность по Шапиро-Уилку и Колмогорову-Смирнову), пропуски, корреляции, обнаружение выбросов методом межквартильного размаха.
- **Проверка качества данных** — 12 правил с тремя уровнями severity (`critical` / `warning` / `info`): пропуски, дисбаланс классов, утечки признаков, низкая дисперсия, дублирование строк и другие.
- **Гибридный рекомендатель типа ML-задачи** — двухслойная архитектура: детерминированные правила (Слой 1, дерево решений с обоснованием) и Random Forest на 16 meta-признаках (Слой 2, обучен на 177 датасетах: 27 реальных + 150 синтетических, CV f1_macro = 0.976 ± 0.022). Между слоями — numeric-discrete bridge, восстанавливающий достижимость ML-слоя для multiclass-кейсов с числовыми метками.
- **Каталог похожих датасетов** — 29 эталонных наборов из sklearn / UCI / GitHub, поиск top-K через косинусную меру на 128-мерных эмбеддингах с HNSW-индексом pgvector.
- **Baseline-обучение** — Logistic / Ridge регрессии и Random Forest (классификация или регрессия в зависимости от target), 5-fold cross-validation, автоматическая фильтрация колонок с подозрением на утечку, важность признаков.
- **PDF-отчёты** — Jinja2-шаблон + WeasyPrint, server-side графики через matplotlib (DejaVu Sans для кириллицы), нумерация страниц через CSS Paged Media, RFC 5987 в Content-Disposition для корректных имён файлов с кириллицей.
- **История анализов** — пагинированный список своих анализов с фильтрацией по статусу, переход к любому анализу или его повторной генерации.
- **Аутентификация и роли** — JWT-токены, два уровня доступа (`user` / `admin`); административная панель со статистикой по системе и списком пользователей, защищена dependency `get_current_admin`.

## Стек

- Backend: Python 3.11, FastAPI, SQLAlchemy 2, Alembic, Pydantic 2
- Frontend: React 19 + TypeScript, Vite, Tailwind CSS, Plotly.js
- Database: PostgreSQL 15 + pgvector
- ML / Reports: scikit-learn, pandas, scipy, matplotlib, WeasyPrint, Jinja2
- DevOps: Docker Compose, Nginx

## Требования

- Docker 24+ и Docker Compose v2
- GNU Make
- Свободные порты: `3000` (фронт)

## Быстрый старт

```bash
# 1. Создать .env из шаблона и подставить криптографический JWT_SECRET
cp .env.example .env
python3 -c "import secrets; print(secrets.token_urlsafe(32))"
# (полученное значение записать в .env как JWT_SECRET)

# 2. Поднять все сервисы
make up

# 3. Применить миграции БД
make migrate

# 4. Загрузить справочники, обучить мета-классификатор и заполнить
#    каталог похожих датасетов
make seed-all

# 5. Открыть приложение
# http://localhost:3000
```

## Команды Makefile

| Команда | Назначение |
|---|---|
| `make up` | Поднять все контейнеры |
| `make down` | Остановить контейнеры |
| `make logs` | Посмотреть логи |
| `make migrate` | Применить миграции БД |
| `make seed-all` | Загрузить справочники, обучить мета-классификатор, заполнить каталог похожих датасетов |
| `make test` | Запустить тесты бэкенда |
| `make clean` | Полная очистка (удаление volumes — БД будет потеряна) |

## Создание администратора

После регистрации первого пользователя через UI можно повысить его до администратора:

```bash
docker compose exec backend python -m app.scripts.promote_to_admin <email>
```

Затем выполнить logout и повторный login в UI — пункт «Админ» появится в шапке навигации.

## Демо-датасеты

Для проверки трёх основных веток рекомендатора подойдут канонические наборы данных, доступные на Kaggle и в `sklearn.datasets`:

- **Titanic** — бинарная классификация (`Survived`).
- **Iris** — многоклассовая классификация (`species`), активирует ML-слой через numeric-discrete bridge.
- **Boston Housing** — регрессия (`medv`).
